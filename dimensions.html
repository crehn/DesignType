Simple vs. Powerful

Simple Powerful
Developers who are in favor of simple solutions ...
  • ... avoid complexity (KISS)
  • ... don't like large amouts of code (MIMC)
  • ... only implement what's necessary (YAGNI)
  • ... tend towards explicit solutions (RoE)
 Developers who are in favor of powerful solutions ...
  • ... like code that solves many problems at once (GP)
  • ... like to identify things that vary and encapsulate them (EVC)
  • ... like generics, frameworks, new technology, abstractions, etc.

Abstract vs. Concrete

Abstract Concrete
Developers who think abstractly ...
  • ... thnk in terms in abstractions, concepts, and models (MP)
  • ... assign responsibilities based on Tell, don't Ask/Information Expert (TdA/IE)
  • ... like code to be separately understandable (PSU)
  • ... keep sure to hide implementation details (IH/E)
 Developers who think concretely ...
  • ... think in code (i.e. in terms of classes methods and statements)
  • ... keep their code clean by eliminating duplication (DRY)
  • ... don't like large amouts of code (MIMC)
  • ... like to identify things that vary and encapsulate them (EVC)

Pragmatic vs. Idealistic

Pragmatic Idealistic
Developers who have a pragmatic attitude ...
  • ... only implement what's necessary (YAGNI)
  • ... tend towards explicit solutions (RoE)
  • ... don't focus on principles like LSP, MP or PSU
  • ... aren't afraid of duplicating code if there is a reason for that
  • ... frequently trade code quality for time to market
 Developers who have an idealistic attitude ...
  • ... strive for principles like MP or PSU, TdA/IE, UP
  • ... avoid duplication whereever they can

Technology vs. Stability

Technology Stability
Developers who embrace new technology ...
  • ... like trying out new frameworks, libraries, design strategies, and languages
  • ... prefer keeping up with the times by frequently updating to new technology
  • ... move fast and don't fear breaking things
  • ... won't be left behind by evolving technology
 Developers who strive for stable and robust technolgy ...
  • ... like stable, proven solutions which stood the test of time
  • ... withstand the hype of new technology which might vanish soon
  • ... don't fear being left behind by evolving technology
